Key Concept — Image as a Matrix

A grayscale image is simply an M×N matrix of numbers. Each number is one pixel's brightness. A color image is M×N×3 — three such matrices stacked, one for each color channel (R, G, B).

Apply the Negative Transformation
Easy Done ▼

Invert a grayscale image using the negative transformation. Dark regions become bright and vice versa. Notice how details that were hard to see in shadow regions become visible after inversion.

Formula
s = (L − 1) − r = 255 − r    (for uint8)

Every pixel's intensity is reflected about the midpoint 127.5. A pixel that was 200 (light) becomes 55 (dark). A pixel that was 20 (near black) becomes 235 (near white).

In uint8 arithmetic, values saturate — they clamp to [0,255]. So uint8(10) - uint8(20) gives 0 not -10. By converting to double first, subtraction works correctly with the full integer range, then we cast back to uint8.
The negative formula s=255-r is a scalar operation on a single intensity value. For RGB, you'd have to apply it separately to each of the 3 channels. Converting to grayscale first simplifies visualization for this lab.

Gamma Correction with Power Law
Easy Done ▼

Apply the power law transformation with three different gamma values (γ = 0.4, 1.0, 2.5) and arrange the results in a single figure. Observe how gamma below 1 brightens the image while gamma above 1 darkens it.

Formula & Rule
s = c · rγ    (r normalized to [0,1], c = 1)

γ < 1 → curve bows upward → dark pixels lifted → brighter image. γ > 1 → curve bows downward → mid-tones pushed down → darker, higher contrast. γ = 1 → identity (no change).

MATLAB arrays start at index 1, but gray levels start at 0. So gray level 0 maps to h(1), gray level 1 maps to h(2), and gray level 255 maps to h(256). The offset of +1 bridges this gap between 0-based intensity values and 1-based MATLAB indexing.
A histogram skewed left (lots of bars near 0) means the image is dark/underexposed. Skewed right means overexposed. A narrow peak in the middle means low contrast.

cumsum([a, b, c, d]) returns [a, a+b, a+b+c, a+b+c+d]. For a normalized histogram that sums to 1, the last element of cumsum will always be exactly 1.0. This running total is the CDF — it tells you the probability that any pixel has intensity ≤ k.
Histogram equalization maximizes contrast by spreading values over the full range. But human perception doesn't always prefer maximum entropy — skin tones may wash out, skies become over-bright. The algorithm is mathematically optimal by information theory, but perceptually it sometimes needs manual fine-tuning afterward.
